# ClickHouse 25.3.14.14 vs 24.3.5.46 (prod) — 2026-07-22

**Verdict: PASS — performance parity with prod.** Workload-weighted ratio **1.039**
(49.0s → 50.9s), within the ≤1.05 bar. First clean version found in the bisect:
24.3 ✅ → **25.3 ✅** → 26.3 ❌ (+19%) → 26.6 ❌ (+14%).

## Environment

- Baseline: v24.3.5.46-lts (exact prod version), port 9000; candidate: v25.3.14.14-lts, port 9010
- Both restored from backup `2026-07-21T20-10-54` (25.3 restore of the 24.3-written backup via
  clickhouse-backup 2.8.0 succeeded — backup-compat gate passed)
- Same setup as prior evaluations (45-query corpus, 16 interleaved rounds; full results:
  [25.3.14.14-report.tsv](25.3.14.14-report.tsv))

## Results

- **Correctness**: result rows identical for all 45 queries, zero failures.
- **`ARRAY JOIN measures` chart queries: parity** (ratios 1.02–1.14; the query that regressed
  to 384 ms on 26.x runs 265→286 ms). All ratios above 1.14 are sub-10 ms queries where 1 ms
  rounding dominates.
- Heaviest daily aggregate slightly faster: 1438→1391 ms (0.97).
- The 2–7× `SELECT DISTINCT` improvements seen on 26.x are **not** present — they land in the
  same window as the ARRAY JOIN regression. Here everything is simply flat.
- Memory: unchanged across the board.

## Caveat

25.3 LTS is at/near end of standard support (LTS = 1 year from 2025-03). It is still a
~2-year fix uplift over 24.3, but 25.8 LTS should be tested next: if clean, it is the newer
safe target; if regressed, the cliff is in (25.3, 25.8] — consistent with
[ClickHouse#91430](https://github.com/ClickHouse/ClickHouse/issues/91430) being reported
against 25.8.
