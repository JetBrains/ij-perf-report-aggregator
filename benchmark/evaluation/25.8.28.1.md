# ClickHouse 25.8.28.1 vs 24.3.5.46 (prod) — 2026-07-22

**Verdict: PASS — recommended upgrade target.** Workload-weighted ratio **0.946**, i.e. 25.8 is
~5% _faster_ than prod on the real dashboard workload. Final bisect picture:
24.3 ✅ → 25.3 ✅ (1.04) → **25.8 ✅ (0.95, best)** → 26.3 ❌ (1.19) → 26.6 ❌ (1.14).
The ARRAY JOIN executor regression landed in **(25.8, 26.3]** — re-evaluate 26.x LTS once
upstream addresses it ([ClickHouse#91430](https://github.com/ClickHouse/ClickHouse/issues/91430)).

## Environment

- Baseline: v24.3.5.46-lts (exact prod version), port 9000; candidate: v25.8.28.1-lts, port 9010
- Both restored from backup `2026-07-21T20-10-54` (25.8 restore of the 24.3-written backup via
  clickhouse-backup 2.8.0 succeeded — backup-compat gate passed)
- Same setup as prior evaluations (45-query corpus, 16 interleaved rounds; full results:
  [25.8.28.1-report.tsv](25.8.28.1-report.tsv))

## Results

- **Correctness**: result rows identical for all 45 queries, zero failures.
- **`ARRAY JOIN measures` chart queries: parity to slightly better** (1.01–1.09; the query that
  hits 384 ms on 26.x runs 266→278 ms here).
- **The two heaviest queries improved**: daily aggregate 1443→1297 ms (0.90) with memory
  164→121 MB; 3-month machine-group aggregate 321→272 ms (0.85) with memory 119→98 MB.
- Part of the `SELECT DISTINCT` speedup already present (68→38 ms machine-distinct); the rest
  (project-distinct 2–7×) only arrives with the regressed 26.x versions.
- All ratios ≥1.1 are sub-20 ms queries where 1 ms rounding dominates.

## Shipping notes (BACKUP.md has the details)

- 25.8 LTS support window ends ~Aug 2026 — it is nonetheless the newest safe version; 26.3 LTS
  is blocked by the ARRAY JOIN regression.
- Rollback insurance: before upgrading, note the latest prod backup name; after 25.8 writes its
  first backup, rolling back to 24.3 requires `RESTORE_BACKUP_NAME` pinned to a pre-upgrade backup.
